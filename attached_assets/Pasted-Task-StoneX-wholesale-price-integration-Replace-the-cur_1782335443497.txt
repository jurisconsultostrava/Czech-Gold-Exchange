Task — StoneX wholesale price integration
Replace the current fixed-price system with a live premium-based pricing engine.
How pricing works:

StoneX WE SELL = premium in EUR above spot that YOU pay to buy the product
Your retail price = spot × weightGrams + StoneX premium + your margin
Your buyback price = spot × weightGrams + StoneX WE BUY premium (negative = below spot)

Step 1 — New database table for StoneX premiums:
prismamodel StoneXPremium {
  id             String   @id @default(cuid())
  productId      String   // matches Product.id (CODE)
  stonexName     String   // original English name from StoneX
  buyPremiumEur  Float?   // WE BUY premium (can be negative)
  sellTier1Qty   Int      @default(1)
  sellTier1Eur   Float    // WE SELL premium for 1+ pcs
  sellTier2Qty   Int?
  sellTier2Eur   Float?
  sellTier3Qty   Int?
  sellTier3Eur   Float?
  sellTier4Qty   Int?
  sellTier4Eur   Float?
  updatedAt      DateTime @updatedAt
}
Step 2 — New pricing calculation in /api/prices:
Current response: {id, priceCzk, purchasePriceCzk, inStock}
New response:
typescript{
  id: string,
  // Retail price = spot × weight + StoneX sell premium tier1 + your margin
  priceCzk: number,
  priceEur: number,
  // Buyback = spot × weight + StoneX buy premium (if exists)
  purchasePriceCzk: number,
  purchasePriceEur: number,
  // Volume tiers (if StoneX has tiers)
  tiers: Array<{ qty: number, priceCzk: number, priceEur: number }>,
  // Deferred = retail × (1 - deferredDiscountPct/100)
  deferredPriceCzk: number,
  deferredPriceEur: number,
  inStock: boolean,
  // Price breakdown for detail page
  breakdown: {
    spotEurPerGram: number,
    metalValueEur: number,
    stonexPremiumEur: number,
    yourMarginCzk: number,
  }
}
Calculation logic:
typescriptconst spotEurPerGram = currentSpot.gold_eur_per_gram  // from spot API
const metalValueEur = spotEurPerGram × product.weightGrams
const stonexPremium = StoneXPremium.sellTier1Eur ?? 0
const marginCzk = getMargin(product)  // from Settings per category
const priceEur = metalValueEur + stonexPremium
const priceCzk = priceEur × eurToCzk + marginCzk

const buyPremium = StoneXPremium.buyPremiumEur ?? 0
const purchasePriceEur = metalValueEur + buyPremium
const purchasePriceCzk = purchasePriceEur × eurToCzk
For products without StoneX premium → fall back to xaumanager XML price (PRICE_VAT).
Step 3 — StoneX PDF parser:
Create scripts/parse-stonex.ts that:

Fetches https://stonexbullion.com/api/client/catalog/pdf/?t=${Date.now()}&url=%2Fen%2Fgold%2F&update_filters=true&metal_ids%5B%5D=1&term=&page=1 (gold) and equivalent URLs for silver (metal_ids[]=2), platinum (metal_ids[]=3), palladium (metal_ids[]=4)
Parses the text PDF response — each line format: {NAME} {WE_BUY_EUR} {WE_BUY_PCT}% {QTY}+ {WE_SELL_EUR} {WE_SELL_PCT}% {WEIGHT}
Matches StoneX NAME to Product by CODE — match logic:

Normalize both names (lowercase, remove punctuation)
Match on weight + series keywords (Maple Leaf, Krugerrand, Britannia etc.)
Store match confidence score, only auto-insert if confidence > 0.85


Upserts StoneXPremium table
Prints report: X matched, Y unmatched, Z updated
